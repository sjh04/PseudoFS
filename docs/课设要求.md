## 6 基本原理

### 6.1 UNIX 文件格式 — 多层抽象

1. **磁盘 → 分区**：一个物理磁盘可划分成分区，每个分区逻辑上看作独立磁盘
2. **分区 → 扇区**：扇区是磁道上的基本存储单元
3. **扇区 → 簇**：将若干扇区合并成一个逻辑块（簇），按簇编号
4. **簇 → 文件系统分区**：簇序列分为超级块、索引节点区和数据块区

### 6.2 文件卷结构

簇序列分为以下 3 部分：

- **超级块**：占用 1# 号块，存放文件系统结构和管理信息
- **索引节点区**：2# ~ (k+1)# 号块，存放索引节点表；每个索引节点大小相同、编号唯一
- **数据区**：(k+2)# ~ n# 号块，存放文件内容

### 6.3 超级块

描述文件系统的状态，包括：
- 磁盘空闲块栈
- 空闲 i 节点栈

### 6.4 i 节点（inode）

- 存放文件说明信息，每项 32 字节
- 分为**磁盘 i 节点**和**内存 i 节点**
- **SFD**（符号文件目录）：存放文件名及指向文件说明信息的标识符 ID
- **BFD**（基本文件目录）：即 i 节点，存放文件说明信息和对应标识符
- 目录表目中仅包含文件名和 i 节点号

#### 内存 i 节点的逻辑组织 — Hash 链表

- 共 128 个 Hash 链表 `hinode[0..127]`
- Hash 函数：`ihash(x) = &hinode[(int)(x) & 128]`
- 队首由 `hinode` 中的 `i_forw` 指出

### 6.5 空闲磁盘块分配与回收 — 成组块链接法

- 把空闲块分成若干组，每组 50 块（第一组 49 块）
- 每组第一个空闲块登记下一组空闲块的块号和空闲块数
- 各组通过链表指针串在一起

**分配算法** `AllocateOneBlock()`：
```
if S[0] == 1 then
    if S[1] == 0 then 分配失败，进程等待
    else r = S[1]; ReadABlock(S, r); return r
else
    r = S[S[0]]; S[0]--; return r
```

**回收算法** `FreeABlock(BlockNo)`：
```
if S[0] == 最大总块数 N then
    write 堆栈S 到块BlockNo中; S[0]=1; S[1]=BlockNo
else
    S[0]++; S[S[0]] = BlockNo
```

### 6.6 混合索引结构

索引数组 `i-addr[i]` 支持四种文件大小：

| 类型 | 大小范围 | 索引方式 |
|------|----------|----------|
| 小型文件 | < 5KB | 直接索引 |
| 中型文件 | 5KB ~ 90KB | 一次间址 |
| 大型文件 | 90KB ~ 14.54MB | 二次间址 |
| 巨型文件 | 14.54MB ~ 1GB | 三次间址 |

### 6.7 FAT16 文件卷格式

```
| 引导块 | FAT1 | FAT2 | 目录及数据区 |
```

## 7 设计方案提示

### 7.1 设计手段

- 申请一定大小的内存空间作为文件系统的空间（`malloc()` 或 `calloc()`）
- 利用 DOS/Windows 文件操作将内存信息写到磁盘文件（虚拟盘）

### 7.2 文件卷组织

- 将文件卷分块，用位示图或成组块链管理
- 块大小：512 字节
- i 节点区：32 块
- 数据区：512 块（参数可根据实际情况设置）

### 7.3 系统结构（三层）

1. **块管理层**：块分配、块回收、块读、块写
2. **文件控制块层**：目录查找 `namei()`、`iget()`、`iput()` 等
3. **命令及命令解释层**：`open()`、`close()` 等

### 7.4 涉及的主要数据结构

- 文件目录结构（BFD 或目录列表文件）
- 存储介质管理方式（位示图、空闲块链接法、成组块链接法、FAT 法）
- 存储介质组织结构（引导块、超级块、索引节点块、数据区）
- 文件逻辑结构（字符流式、记录式）
- 文件物理结构（串联文件、索引文件）

## 8 主要数据结构

### 8.1 超级块结构

- i 节点块数、空闲 i 节点数、空闲 i 节点数组、空闲 i 节点指针、铭记 i 节点
- 数据块块数、空闲块块数、空闲块数组、空闲块指针
- 超级块修改标志

### 8.2 i 节点结构

- **磁盘 i 节点**：关联文件数、存取权限、用户 ID、文件大小、存放文件的物理块号
- **内存 i 节点**：磁盘 i 节点内容 + 两个 i 节点指针 + 磁盘 i 节点标志 + 引用计数

### 8.3 其他数据结构

- **打开文件结构**：文件操作标志、引用计数、指向内存 i 节点的指针、文件读写指针
- **用户结构**：用户类别、用户 ID、组 ID、用户打开文件表
- **目录结构**：目录项数组 + 目录大小
- **文件物理结构**：索引文件

### 8.4 头文件常量

```c
#define BLOCKSIZ      512     // 每块大小
#define SYSOPENFILE   40      // 系统打开文件表最大项数
#define DIRNUM        128     // 每个目录最大目录项数
#define DIRSIZ        14      // 目录项名字部分字节数（另加 i 节点号 2 字节）
#define PWDSIZ        12      // 口令字
#define PWDNUM        32      // 最多 32 个口令登录
#define NOFILE        20      // 每个用户最多打开 20 个文件
#define NADDR         10      // 每个 i 节点最多指向 10 块 (addr[0]~addr[9])
#define NHINO         128     // 128 个 Hash 链表（必须为 2 的幂）
#define USERNUM       10      // 最多允许 10 个用户登录
#define DINODESIZ     32      // 每个磁盘 i 节点所占字节
#define DINODEBLK     32      // 所有磁盘 i 节点共占 32 个物理块
#define FILEBLK       512     // 共 512 个目录文件物理块
#define NICFREE       50      // 超级块中空闲块数组最大块数
#define NICINOD       50      // 超级块中空闲节点最大块数
#define DINODESTART   (2 * BLOCKSIZ)              // i 节点起始地址
#define DATASTART     ((2 + DINODEBLK) * BLOCKSIZ) // 目录/文件区起始地址
```

## 9 主要函数

| 函数 | 功能 |
|------|------|
| `ialloc()` / `ifree()` | 磁盘 i 节点的分配与释放 |
| `iget()` / `iput()` | 内存 i 节点的获取与释放 |
| `access()` | 判别用户对文件的访问权限（read/write/execute） |

## 10 主控程序基本流程

1. 进入模拟文件系统 VFS
2. 是否需要格式化 VFS？
3. 若是，则格式化 → 转 6
4. 否则，文件系统文件是否存在？若在 → 转 7，否则：
5. 申请内存，格式化文件系统（组织存储介质、划分空间等）→ 转 6
6. 初始化相关数据（结构）
7. 显示命令列表 / 命令提示符 / GUI 界面
8. 选择命令及输入参数 / 接受命令
9. 若是退出命令 → 转 12
10. 执行相关操作
11. 显示结果 / 转 7
12. 作退出后的保存工作
