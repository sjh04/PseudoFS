- **今日任务目标：** 双引擎核心打通。UNIX FS 完成格式化、inode 分配/回收、混合索引寻址与目录解析，`UnixFs` 完整实现 `IFileSystem`；FAT16 完成格式化、FAT 表初始化与簇链分配；Shell 基础命令对接 VFS 跑通；用户管理类实现；TUI 四面板渲染与输入循环跑通。
- **已完成模块：**
  - UNIX FS：`fs_format`（初始化超级块、建立成组空闲块链、空闲 inode 栈、创建根目录）
  - `InodeManager`：`ialloc` / `ifree` / `iget` / `iput`
  - `SuperBlock`：`balloc` / `bfree`（成组块链接法）
  - `bmap` 混合索引（直接 + 一次间址 + 二次间址）块号转换
  - `DirectoryManager`：`namei` 路径解析（绝对/相对）
  - FAT16：引导扇区写入与格式化、FAT1/FAT2 双表初始化、簇链分配/回收、8.3 目录项 CRUD
  - Shell：`format` / `pwd` / `cd` / `ls` / `mkdir` / `rmdir` / `touch` 对接 VFS 跑通
  - 用户管理：`UserManager`（add / auth / getById）
  - TUI：四面板（目录树/磁盘状态/终端/功能键栏）布局 + 输入循环渲染
- **进行中模块：**
  - 文件 `open` / `read` / `write` / `close` 全链路（需打开文件表联动）
  - FAT16 `fs_write` 大文件增量写入（当前一次性整块写，大文件越界）
  - 访问权限检查尚未接入读写路径
  - main.cpp 命令集成、TUI 接入真实 FS 数据
- **遇到的问题：**
  - 二次间址回收时索引块本身也需释放，最初只回收数据块导致索引块泄漏
  - 成组链最后一组不足 50 块的边界处理（组内块数 `nfree` 初值）
  - FAT16 簇号与物理块号的偏移换算（数据区起始块 +（簇号−2））
  - 程序异常退出时内存中脏超级块未落盘 → 数据丢失，引出 auto-sync 讨论
