# PseudoFS 全流程手动测试文档

> 目的：跟着从上到下敲一遍，就能**亲手验证每一项功能**、熟悉命令、并清楚知道「我们实现了什么 / 没实现什么」。
> 每个测试块给出：**命令** → **预期输出** → **验证点** → 对应**需求编号**（见 `docs/项目需求.md`）。
> 约定：行首 `pfs$` 代表程序提示符（实际是当前路径，如 `/home/alice$`）；**「(成功无输出)」表示命令成功时不打印任何东西**，只回到提示符。

---

## 0. 准备

### 0.1 构建
```bash
cd PseudoFS && mkdir -p build && cd build
cmake .. && make -j$(nproc)
```

### 0.2 运行模式
```bash
./pfs            # CLI，UNIX 引擎（默认）
./pfs --fat16    # CLI，FAT16 引擎
./pfs --tui      # ncurses 四面板界面
./pfs --format   # 强制格式化（清空并重置用户表）
./pfs_tests      # 跑 245 个自动化单元测试（应全过）
```

### 0.3 关键须知（影响你的测试预期）
- **启动即自动登录为 root**（uid=0），可直接建用户/改文件。`logout` 后才需重新 `login`/`su`。
- **读写文件是 fd 流程**：`open` 返回 fd → `read <fd>` / `write <fd> <text>` → `close <fd>`。
- 持久化文件（与可执行文件同目录，想从干净状态开始就删掉它们或用 `--format`）：
  | 文件 | 内容 |
  |------|------|
  | `pfs_unix.img` / `pfs_fat16.img` | CLI 模式两引擎的磁盘镜像 |
  | `pfs_tui_unix.img` / `pfs_tui_fat16.img` | TUI 模式两引擎的磁盘镜像（独立，防 F2 切换互踩） |
  | `pfs_users.dat` | 用户表（跨引擎、跨模式共享） |
  | `pfs_ops.log` | 操作日志（供 `log`/`replay`） |
- **建议**：每开始一大节测试前，用 `format` 命令或重启带 `--format`，从干净状态开始，结果才好对照。

[ ] touch /secret.txt

[ ] close 0

## 5. 文件操作（E-01~E-06 / B-02）

干净启动：

### 5.1 创建 / 打开 / 写 / 读 / 关
```
[ ] touch note.txt                    → (成功无输出)         (E-01)
[ ] open note.txt rw                  → fd=0                  (E-02，返回描述符)
[ ] write 0 line-one                  → 8 bytes written       (E-04)
[ ] close 0                           → (成功无输出)         (E-05)
[ ] open note.txt r                   → fd=0
[ ] read 0 4                          → line                  (E-03，读前 4 字节)
[ ] read 0                            → -one                  (偏移连续，读剩余)
[ ] close 0
[ ] cat note.txt                      → line-one              (B-02)
```

### 5.2 追加 vs 覆盖（理解三种打开模式）
```
[ ] open note.txt a                   → fd=0                  (a=追加，偏移在文件尾)
[ ] write 0 -appended                 → 9 bytes written
[ ] close 0
[ ] cat note.txt                      → line-one-appended     （追加到尾部）
[ ] open note.txt w                   → fd=0                  (w=覆盖，打开即清空★)
[ ] write 0 fresh                     → 5 bytes written
[ ] close 0
[ ] cat note.txt                      → fresh                 （旧内容被清，无残留尾巴★）
[ ] stat note.txt                     → Size: 5 bytes
```
**验证点**：`r`/`w`/`rw`/`a` 四模式；`w` 会截断旧内容（O_TRUNC），`rw` 不截断（覆盖但保留更长的尾部）。

### 5.3 分页查看 B-02
分页器**每屏 20 行**，只有内容 **> 20 行** 才会弹 `--More--` 提示；且**仅在真实终端**生效（管道/重定向会一次性 dump）。用双引号 + `\n` 转义造一个多行文件来测：
```
[ ] more note.txt                     → fresh（仅 1 行 ≤20，直接显示就返回，不分页 —— 正常）
[ ] touch m.txt
[ ] open m.txt rw
[ ] write 0 "l01\nl02\nl03\nl04\nl05\nl06\nl07\nl08\nl09\nl10\nl11\nl12\nl13\nl14\nl15\nl16\nl17\nl18\nl19\nl20\nl21\nl22\nl23\nl24\nl25"
                                       → 99 bytes written（\n 在双引号里是真换行）
[ ] close 0
[ ] cat m.txt                         → 25 行
[ ] more m.txt                        → 显示 20 行 + 反白 --More-- (20/25) [Enter=more, q=quit]
                                          回车→看剩余 5 行；q→退出
```
> 退出键是 **q**（不是 `/q`），翻页是 **回车**。在 shell 提示符下敲 `/q` 会得到 `Unknown command: /q`（因为那是命令行，不是分页器内）。

### 5.4 删除 E-06
```
[ ] rm note.txt                       → (成功无输出)
[ ] ls                                → note.txt 没了
[ ] mkdir tree1
[ ] touch tree1/inner.txt
[ ] rm tree1                          → 失败（rm 不删目录，需 -r）
[ ] rm -r tree1                       → (成功无输出)          (E-06 递归删除)
[ ] ls                                → tree1 没了
```

### 5.5 中文 / UTF-8 支持
文件名和内容都是字节透明存储（UTF-8），终端为 UTF-8 即可显示中文。
```
[ ] mkdir 文档                         → (成功无输出)
[ ] touch 文档/笔记.txt
[ ] open 文档/笔记.txt rw
[ ] write 0 你好世界                   → 12 bytes written（4 字 ×3 字节）
[ ] close 0
[ ] cat 文档/笔记.txt                  → 你好世界
[ ] ls 文档                            → 笔记.txt
```
**说明 / 边界**：
- **CLI**：开箱即用（字节透明 + UTF-8 终端）。
- **TUI**：需链接宽字符版 `ncursesw`（`CMakeLists.txt` 已设 `CURSES_NEED_WIDE`）+ `setlocale(LC_ALL,"")`（已调用）。目录树/终端面板能正确显示与输入中文。
- **文件名上限 14 字节** → 中文名最多约 **4 个汉字**（课设 spec 固定 `MAX_FILENAME=14`，超出会截断成乱码尾巴）。
- TUI 里删一个汉字需按 **3 次退格**（按字节删）；`ll`/面板列对齐按字节算，中文名可能略错位（仅视觉，功能正常）。
- 终端环境需 UTF-8（`echo $LANG` 含 `UTF-8`；SSH/终端用 UTF-8 编码）。

## 6. 混合索引 / 大文件（A-02）★验收重点

UNIX 引擎用 inode 的 `di_addr[10]` 地址数组做**混合索引**。每个间址块装 `512/2 = 256` 个块号：

| 地址槽 | 寻址方式 | 单独覆盖范围 | 触发文件大小 |
|--------|---------|------------|------------|
| `addr[0..5]` | 直接索引 | 6×512 = 3 KB | 0 起 |
| `addr[6]`、`addr[7]` | 一次间址 | 2×256×512 = 256 KB | > 3 KB |
| `addr[8]` | 二次间址 | 256×256×512 = 32 MB | **> 259 KB** |
| `addr[9]` | 三次间址（预留） | — | 未实现 |

**理论寻址上限 ≈ 32 MB**；但默认虚拟盘只有 512 数据块 = **256 KB**，所以默认盘下单文件最大约 252 KB（只用到直接+一次间址）。要完整展示二次间址，需把盘临时调大（见 §6.2）。

### 6.1 默认盘：跨「直接索引 → 一次间址」（~12 KB 文件）

单次 `write` 一行内容有限，用**多次追加**堆出大文件。一键脚本（在 `build/` 目录跑）：
```bash
{ echo touch big.bin; for i in $(seq 12); do echo "open big.bin a"; \
  echo "write 0 $(python3 -c 'print("X"*1000)')"; echo "close 0"; done; \
  echo "stat big.bin"; echo "disk"; echo exit; } | ./pfs --format
```
```
[ ] 预期 stat   → Size: 12000 bytes
[ ] 预期 disk   → Blocks 使用数明显上升（约 31 块）
```
**验证点 A-02**：文件 > 3072 字节（6 个直接块）后开始用一次间址块 `addr[6]`，证明 `bmap_alloc` 的间址分配正确。
> 想验证读回一致：把脚本里 `print("X"*1000)` 换成有规律的内容，再 `open big.bin r` + `read 0 12000` 对照。

### 6.2 完整展示二次间址（临时把盘调大）★

二次间址要文件 > 259 KB，超过默认 256 KB 盘容量，所以先把数据区调大、重建，跑完再还原。

**步骤 1 — 改盘容量并重建**
```bash
# 编辑 include/core/constants.h：
#   constexpr uint32_t DATA_BLK_NUM = 512;   ->   = 8192;   (变成 4 MB 盘)
cmake --build build --target pfs -j$(nproc)
```

**步骤 2 — 建一个 700 KB 文件（远超 259 KB 阈值）并读回**
```bash
cd build
python3 - <<'PY'
import subprocess
TOTAL, CHUNK = 700000, 50000
exp = ''.join(str(i % 10) for i in range(TOTAL))          # 有规律、可校验
cmds = ["touch huge"]
for k in range(0, TOTAL, CHUNK):
    cmds += ["open huge a", "write 0 " + exp[k:k+CHUNK], "close 0"]
cmds += ["stat huge", "disk", "open huge r", f"read 0 {TOTAL}", "close 0", "exit"]
out = subprocess.run(["./pfs","--format"], input="\n".join(cmds)+"\n",
                     capture_output=True, text=True).stdout
size = [l for l in out.splitlines() if "Size:" in l]
print(size[-1] if size else "no size")
print("read-back 逐字节一致:", exp in out)
PY
```
```
[ ] 预期输出 → Size: 700000 bytes
[ ]          → read-back 逐字节一致: True
```

**步骤 3 — dump 该文件的 inode，亲眼看到三级索引全部用上**
```bash
python3 tools/inode_dump.py build/pfs_unix.img
```
预期（块号每次可能不同，关键看哪几级 `<used>`）：
```
inode #48  type=file  mode=0o777  nlink=1  uid=0  size=700000 bytes
  di_addr[10] (0 = unused / hole):
    addr[0..5]  direct[..]            =  ....   <used>     ← 直接索引
    addr[6]     single-indirect[6]    =  ....   <used>     ← 一次间址
    addr[7]     single-indirect[7]    =  ....   <used>     ← 一次间址
    addr[8]     DOUBLE-indirect[8]    =  ....   <used>     ← 二次间址 ★
    addr[9]     triple(reserved)[9]   =     0   <unused>   ← 三次间址（未实现）
  double-indirect addr[8]=#....: 4 single-indirect block(s) -> 850 data block(s) (435200 bytes ...)
  index levels in use: direct + single-indirect + double-indirect
```
```
[ ] 检查点：addr[0..8] 均 <used>，addr[9] <unused>  → 二次间址真实生效，三级索引完整
```

**步骤 4 — 还原回课设规定的 512 块（273 KB）并验证**
```bash
# 把 constants.h 改回 DATA_BLK_NUM = 512
cmake --build build -j$(nproc)
./build/pfs_tests          # 应仍是 [ PASSED ] 245 tests
```

**说明（实现边界）**：二次间址（`addr[8]`）已实现，**在调大盘后实测通过**（§6.2）；默认 256 KB 盘容量下不会触发。三次间址（`addr[9]`）按需求是可选项，**未实现**（预留）。

## 7. 链接（A-04）

干净启动（UNIX 引擎）：
```
[ ] touch orig.txt
[ ] open orig.txt rw
[ ] write 0 shared-data               → 11 bytes written
[ ] close 0
[ ] ln orig.txt hard.txt              → (成功无输出)          (A-04 硬链接)
[ ] stat orig.txt                     → Links: 2              （两个名字共享一个 inode）
[ ] stat hard.txt                     → Links: 2
[ ] cat hard.txt                      → shared-data
[ ] ln -s orig.txt soft.txt           → (成功无输出)          (A-04 软链接)
[ ] readlink soft.txt                 → orig.txt              （软链接存的是目标路径）
[ ] cat soft.txt                      → shared-data           （跟随软链接）
[ ] ll                                → soft.txt@ -> orig.txt （l 开头，@ 标记）
[ ] rm orig.txt                       → (成功无输出)
[ ] cat hard.txt                      → shared-data    ★硬链接仍可读（数据存活，nlink 2→1）
[ ] cat soft.txt                      → cat: soft.txt: broken symbolic link  ★软链接悬空
```
**验证点 A-04**：硬链接共享 inode、删一个名字数据不丢；软链接是独立文件存目标路径、目标删了就悬空。
**实现边界**：链接仅 **UNIX 引擎** 支持；FAT16 的 `ln`/`ln -s`/`readlink` 返回失败（见 §10）。

## 8. 高级文件命令（A-03 / B-01 / B-03 / B-06）

```
[ ] touch src.txt
[ ] open src.txt rw
[ ] write 0 copy-me                   → 7 bytes written
[ ] close 0
[ ] cp src.txt dst.txt                → Copied.               (A-03)
[ ] cat dst.txt                       → copy-me
[ ] mv dst.txt renamed.txt            → Moved.                (B-01 改名/移动)
[ ] ls                                → renamed.txt 在，dst.txt 没了
[ ] stat src.txt                      → File:/Type:/Mode:/UID:GID:/Size:/Links:  (B-03)
[ ] mkdir sub
[ ] mv renamed.txt sub                → mv: cannot create dest（目标是已存在目录，安全报错）
[ ] mv renamed.txt sub/renamed.txt    → Moved.（移动进目录要写出完整目标路径）
[ ] find / src.txt                    → /src.txt              (B-06 按名搜索，返回全路径)
[ ] find / nonexist                   → (无匹配则无输出)
```
**实现边界 B-01 / 重要**：
- `mv`/`cp` 的目标必须是一个**新文件名**或目录内的完整路径（如 `sub/renamed.txt`）。直接 `mv 文件 目录名` **不会**自动放进目录，而是安全报错 `cannot create dest`（不像真实 mv 那样自动 dir/basename —— 该便利特性未实现）。
- `mv` 仅支持**文件**；对目录源会提示 `mv: directory move not supported (use cp+rm)`（目录移动未实现）。
- 注：早期版本里 `mv 文件 已存在目录名` 会**覆写目录数据块、损坏目录**（随后 `find`/`ls` 可能段错误）——此 bug 已修复（`fs_open` 现在拒绝把目录当文件打开）。

## 9. 通配符 / 帮助 / 日志（C-02 / C-04 / C-05 / B-07）

### 9.1 通配符 C-02
> 注：本 shell **不支持 `;` 分隔多条命令**（`;` 会被当成普通参数）。一行一条命令；但 `touch`/`rm`/`ls`/`cat`/`stat` 等支持一条命令带多个文件名。
```
[ ] touch a.txt b.txt c.log           → 一次创建 3 个文件（touch 支持多文件）
[ ] ls                                → a.txt b.txt c.log 都在
[ ] ls *.txt                          → a.txt  b.txt          （只列 .txt）
[ ] rm *.txt                          → (成功无输出)
[ ] ls                                → 只剩 c.log            （glob 只删匹配的）
```
**实现边界**：通配符只对**路径最后一段**展开（`docs/*.log` 可，`a*/b` 不展开）；多条命令请分行敲（无 `;`/`&&` 串联）。

### 9.2 帮助 C-04
```
[ ] help                              → 列出全部 33 条命令及用法
[ ] mkdir -h                          → Usage: mkdir [-p] <path> — create directory
[ ] ls --help                         → 该命令的用法
```

### 9.3 操作日志与回放 C-05
```
[ ] log                               → 列出已执行操作（含时间/用户/命令），自动跳过 help/log/exit
[ ] log clear                         → 清空日志
[ ] (做几个操作如 mkdir lll / touch m.txt)
[ ] replay                            → 重新按序执行日志里的每条操作
```

## 10. 双引擎 FAT16（A-01）

用 `./pfs --fat16 --format` 启动，把 §4/§5/§8 的目录与文件命令再跑一遍，应当同样工作。重点关注**与 UNIX 的差异**：

```
[ ] mkdir /docs
[ ] cd /docs
[ ] touch report.txt
[ ] open report.txt rw
[ ] write 0 fat16-content             → 13 bytes written
[ ] close 0
[ ] cat report.txt                    → fat16-content
[ ] ls /docs                          → REPORT.TXT  ★名字变大写（8.3 规范）
[ ] cp report.txt copy.txt            → Copied.               （多簇/簇链分配）
[ ] disk                              → FS: FAT16 ; Inodes 恒为 0/0（FAT 无 inode）
```
**FAT16 与 UNIX 的差异（都是 FAT 规范行为，非 bug）**：
| 命令 | UNIX | FAT16 |
|------|------|-------|
| 文件名 | 原样 14 字符 | **强制大写 8.3**（`report.txt`→`REPORT.TXT`） |
| `chmod` | 改 rwx 权限 | **空操作**（FAT 无权限位） |
| `ln` / `ln -s` / `readlink` | 支持 | **不支持**（返回失败） |
| 权限拒绝 | 生效 | 无（FAT 无属主/权限） |
| `disk` 的 Inodes | 真实计数 | 恒 0/0 |
| 索引方式 | inode 混合索引 | FAT 簇链 |

**F2 切换（仅 TUI）**：见 §11。

## 12. 边界 / 异常测试（健壮性，验收加分）

```
[ ] cat 不存在的文件                  → cat: ...: no such file or directory
[ ] open 不存在的文件 r               → open: no such file or directory
[ ] read 99（无效 fd）                → 报错，不崩溃
[ ] close 99（无效 fd）               → 报错，不崩溃
[ ] cd /no/such/path                  → cd: no such directory
[ ] rmdir 非空目录                    → not empty or not found
[ ] useradd（非 root 时）             → useradd: failed (need root, ...)
[ ] 乱敲 asdfgh                        → Unknown command: asdfgh（不崩溃）
[ ] write 一个超长行（>2000 字符）     → 完整写入，size 正确（输入不再被截断★）
[ ] 把磁盘写满（持续 append 大文件）   → Blocks 到上限，写入优雅失败，不崩溃
[ ] 输入一半按 Ctrl+C                   → 显示 ^C，丢弃当前行，刷新提示符；程序不退出★
[ ] 空提示符下按 Ctrl+C                 → 换行重新提示，不退出（要退出用 exit 或 Ctrl+D）
```
**验证点**：所有非法输入都被捕获并给出错误信息，进程**不崩溃**；`Ctrl+C` 取消当前输入行而非杀死程序（像 shell），`Ctrl+D`/`exit` 才退出。

## 14. 已知设计取舍 / 边界（验收会被追问，先背好）

1. **空闲 inode 栈耗尽不回扫磁盘**：超级块只缓存 50 个空闲 inode 号，用完后 `ialloc` 直接返回失败（未实现回扫补充，代码注释标了 TODO）。实际可用文件数受此影响。
2. **目录的硬链接计数恒为 1**：未维护经典的「2 + 子目录数」，`ll` 里目录链接数显示 1。属 cosmetic，`rmdir` 靠判空而非靠计数。
3. **每次写操作都整盘落地**（`sync` 保存整个镜像）：稳但慢，换取防崩溃丢数据。
4. **密码明文存储**（`pfs_users.dat`）：课设范围可接受，报告应注明真实系统应哈希。
5. **FAT16**：根目录固定 16 项上限；文件名大写 8.3；簇分配是线性扫描 O(n)。
6. **符号链接**相对目标从当前目录解析；带 16 跳环路上限防死循环。
7. **目录项删除是逻辑删**（留空槽复用），目录文件不收缩。
