## A(40%)— UNIX 内核,项目的「心脏」

全场最硬的部分,所有刁钻追问主要落在你这。

| 讲什么 | OS 概念 | 源码 |
|--------|---------|------|
| 块设备层 + 持久化(磁盘 = 内存数组,整盘 dump) | 块设备抽象 | `block_device.cpp` |
| **UNIX 磁盘布局**(必画图 #1) | 文件卷组织 | 学习指导 §4.1 |
| **成组块链接法** balloc/bfree(必画图 #2)★ | 空闲块管理 | `super_block.cpp:99 / :132` |
| **inode 混合索引** bmap(必画图 #3,Q4 答案)★ | 文件寻址 | `inode.cpp:174 / :212` |
| inode 生命周期 iget/iput、`i_count` vs `di_nlink` | 引用计数 / 删除语义 | `inode.cpp:60 / :83` |
| namei 路径解析 | 命名 / 路径 | `directory.cpp:183` |

**讲稿提纲(口述顺序)**
1. 「磁盘就是一段 calloc 的字节数组,所有块操作都是 `blk*512` 偏移 memcpy。」
2. 画布局图:boot[0] / super[1] / inode[2..33] / data[34..545]。
3. 成组链:画「内存 50 格栈 + 磁盘组链」,讲 balloc 两个边界(栈剩 1 项读磁盘组、栈满写回)。
4. 混合索引:照寻址表口述 N<6 直接 / [6][7] 一次 / [8] 二次,算出 3KB + 256KB + 32MB(Q4)。
5. iget/iput:区分两套计数,讲清「nlink==0 且 i_count==0 才真正销毁」。
6. namei:逐段 get → 确认目录 → lookup → put;namei_parent 为何返回父目录。

**你必接的刁钻追问**(必答到位)
- rm 一个还 open 着的文件会怎样? → 只 unlink + nlink--;i_count>0 不回收,close 后才 truncate+ifree。
- `di_addr[i]==0` 会和真实块号 0 冲突吗? → 不会,format 预留数据区前 3 块,balloc 永不发 0。
- 删大文件只放直接块会怎样? → 间址块全泄漏,truncate 必须逐级遍历。
- 成组链能 O(1) 判断第 100 块是否空闲吗? → 不能,要遍历整条链。
- inode 缓存满 / 空闲 inode 栈空? → cache_alloc 返回 null;ialloc 返回 INVALID_BLK 且当前不回扫磁盘(已知 TODO)。

**你必接的刁钻追问**

## C(10%)— 打开文件表(UNIX fd 机制)

把 **系统/用户双层打开文件表** 整块给 C,自包含、好讲,且 C 写过 `test_open_file_table.cpp` 有基础。
源码 `open_file_table.cpp`,学习指导 §6.2。

**讲稿提纲(2~3 句话讲透)**
1. 两层结构:用户表 `ofile[fd]=sys_idx`(私有小整数 fd)→ 系统表(全局 40 项,存 f_flag / f_count / f_inode_no / **f_offset**)。
2. **高光句**:为什么读写偏移 `f_offset` 存在系统表? → 让多个 fd 能共享同一个打开文件的偏移和模式(UNIX dup/共享语义)。
3. alloc_fd:用户表找空 fd → 系统表找 `f_count==0` 项 → 关联;logout 时 close_all 释放该用户全部 fd。

**可能追问**:fd 和系统表项是一对一吗? → 不是,多个 fd 可指向同一系统表项(共享偏移),所以才分两层。

## D(10%)— 格式化 + 访问控制

两个小而经典的 OS 点。

**① 必答 Q3「格式化做了什么」**(源码 `unix_fs.cpp:25`)
清零盘 → 建超级块 + **成组空闲块链** → 建空闲 inode 栈 → 建根目录(写 `.` / `..`)→ 预置 `/etc` `/home`。
一句话:format = 把一块全 0 的「裸盘」初始化成有元数据、有根目录、可挂载的文件卷。

**② rwx 三级访问控制**(学习指导 §4.7)
root 直接放行;否则按身份选三组权限位之一(owner→PERM_U / 同 gid→PERM_G / 其它→PERM_O);O_READ 需 r 位、O_WRITE/O_APPEND 需 w 位。

**兼职**:D 写了报告,顺手讲 **设计权衡 / 已知局限**(学习指导 §11.4):密码明文存储、sync 每次写全盘落地(稳但慢)、空闲 inode 栈耗尽不回扫磁盘。「知道自己的局限」是答辩加分点。

**可能追问**:root 为什么能绕过权限? → check_access 里 `cur_uid==0` 直接 return 放行,符合 UNIX 超级用户语义。
