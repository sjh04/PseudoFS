# PseudoFS 现场演示脚本(主操作手:A)

> 全部命令已用 `build/pfs` **实测跑通**,下方「→」后是**真实输出**,照着念即可。
> 设计目标:8 分钟内覆盖所有评分点(目录 / 文件 I/O / 打开文件表 / 权限 / 链接 / cp-mv / 多用户访问控制 / 持久化 / 双引擎+TUI)。
> 每个 Act 末标注它**证明了哪个功能点 / 必答题**。

---

## 0. 开演前检查(60 秒,务必先做)

```bash
cd PseudoFS/build && make -j$(nproc)        # 确认能编出 pfs
ls pfs                                       # 可执行在
```
- 终端拉大到至少 **100×30**(TUI 四面板才不挤)。
- 干净开场:演示用 `./pfs --format`(强制格式化,从全 0 裸盘开始)。
- **持久化那一幕(Act 6)反过来——不要加 `--format`**,否则盘被重置就演示不出"重启数据还在"。

> ⚠️ 三个最易翻车点,先记牢:
> 1. **`open` 返回的 fd 是 `0`,不是 3**。后面 `write`/`close` 跟着屏幕上的真实数字念。
> 2. **写完别用同一个 fd 去 `read`**——偏移已在文件末尾,读不到内容。要么 `close` 后 `cat`,要么重新 `open ... r` 再 `read`。
> 3. **Act 5 的 `permission denied` 是预期结果,不是 bug**——要主动说"看,权限系统把它拦住了"。

## Act 0 — 启动

```
./pfs --format
```
→ `PseudoFS v2.0 [UNIX] — type 'help' for commands, 'exit' to quit`
→ 提示符 `/$`

**话术**:「默认 UNIX inode 引擎,启动即自动以 root 登录,当前在根目录 `/`。」
**证明**:F-01 format · F-02 系统初始化。

## Act 1 — 目录操作

```
pwd
mkdir -p /home/alice
tree
cd /home/alice
pwd
```
→ `pwd` 显示 `/`
→ `tree` 打出 `/home/alice` 多级结构
→ `cd` 后 `pwd` 显示 `/home/alice`

**话术**:「`mkdir -p` 一次建多级目录,底层是 namei 逐段解析路径;`cd` 支持绝对/相对路径和 `~`。」
**证明**:D-01 mkdir(-p) · D-03 cd · D-06 pwd · D-07 tree。

## Act 2 — 文件 I/O + 打开文件表(C 的内容,现场你来演)

```
touch notes.txt
open notes.txt rw
write 0 Hello PseudoFS -- OS course demo
close 0
cat notes.txt
```
→ `open` → **`fd=0`**(念这个数字!)
→ `write 0 ...` → `32 bytes written`
→ `cat` → `Hello PseudoFS -- OS course demo`

**话术**:「`open` 在打开文件表分配一个 fd(这里是 0),`write` 通过它定位 inode、按混合索引把数据落到数据块,`close` 释放 fd。」
**(可选)演示纯读路径**:
```
open notes.txt r
read 0
close 0
```
→ `read 0` → `Hello PseudoFS -- OS course demo`
**证明**:E-01 touch · E-02 open · E-03 read · E-04 write · E-05 close · 打开文件表。

## Act 3 — 权限(rwx 访问控制)

```
ll
chmod 600 notes.txt
ll
```
→ 第一次 `ll`:`-rwxrwxrwx  1 root  32 ...  notes.txt`
→ `chmod 600` 后 `ll`:`-rw-------  1 root  32 ...  notes.txt`

**话术**:「权限位实打实存在 inode 的 mode 里;`chmod 600` 后 owner 只读写、group/other 全无——下一幕就用它演示真正的访问拦截。」
**证明**:A-05 文件访问控制 · C-01 chmod · D-05 ll。

---

## Act 4 — 链接(UNIX 引擎亮点,FAT16 做不到)

```
ln notes.txt hard.txt
stat notes.txt
ln -s notes.txt soft.txt
readlink soft.txt
ls
```
→ `stat notes.txt` → `Links: 2`(硬链接让 nlink 变 2)
→ `readlink soft.txt` → `notes.txt`
→ `ls` → `notes.txt  hard.txt  soft.txt@`(`@` 标记软链接)

**话术**:「硬链接 `hard.txt` 和 `notes.txt` **指向同一个 inode**,所以 nlink=2、共享数据;软链接 `soft.txt` 是另一个 inode,里面存的是目标路径字符串。这正是 UNIX 引擎比 FAT16 强的地方——FAT16 不支持链接。」
**证明**:A-04 硬/软链接 · B-03 stat。

## Act 5 — 复制 / 移动 / 磁盘用量

```
cp notes.txt copy.txt
mv copy.txt renamed.txt
ls
disk
```
→ `cp` → `Copied.` · `mv` → `Moved.`
→ `ls` → `... renamed.txt`
→ `disk` → `FS: UNIX` / `Blocks: 10/512 used` / `Inodes: 9/256 used`

**话术**:「`disk` 实时显示成组链管理下的块用量和 inode 用量。」
**证明**:A-03 cp · B-01 mv · B-04 磁盘可视化(数据侧)。

---

## Act 6 — 多用户 + 访问控制拦截(全场高光,务必演)

```
useradd alice alice 1000 1000
su alice alice
cat /home/alice/notes.txt
su root root
```
→ `useradd` → `User alice created.`
→ `su alice alice` → `Switched to alice`
→ `cat ...` → **`cat: /home/alice/notes.txt: permission denied`**
→ `su root root` → `Switched to root`

**话术**:「`notes.txt` 是 root 的、权限 600。切到 alice 后再读它——**被拒绝**。这证明权限检查真的在 inode 层生效:alice 既不是 owner、也不同 gid,落到 other 位、无 r 权限。切回 root,root 绕过一切检查,又能读了。」
**证明**:U-03 useradd · B-05 su · A-05 权限检查(真实拦截 + root 绕过)。

## Act 7 — 持久化(必考 F-03,反着不要 --format)

```
exit
```
→ `Disk saved. Goodbye.`(整盘 dump 到 `pfs_unix.img`)

```
./pfs                 # 注意:不加 --format!
cat /home/alice/notes.txt
```
→ 重启后 `cat`(此时是 root)→ `Hello PseudoFS -- OS course demo`

**话术**:「退出时把整块虚拟磁盘 dump 成 `pfs_unix.img`;重启**不格式化**,文件系统从镜像恢复,数据原样还在。这就是持久化。」
**证明**:F-03 持久化。

## Act 8 — 双引擎 + TUI(B 协同;最有视觉冲击,放压轴)

```
exit
./pfs --tui
```
TUI 里依次按:
- 看四面板:**目录树** / **磁盘用量条** / **终端** / 标题栏时钟。
- **F2** → 切到 FAT16 引擎(标题栏 `[UNIX]` 变 `[FAT16]`)。在 FAT16 下 `mkdir`/`touch`/`write`/`cat` 同样跑通——**同一套命令、两种底层实现**(VFS 多态)。
- **F3** → 全屏磁盘块位图(红=已用 / 青=元数据 / 绿=空闲)。
- **F5** 刷新 · **F10** 退出。

**话术**:「F2 切换底层从 inode 体系换成 FAT 表,上层命令一行没改——靠的是 `IFileSystem` 抽象接口。两个引擎各有独立磁盘镜像,互不踩坏。」
**证明**:A-01 双引擎 · A-02/A-06 TUI · B-02 磁盘可视化 · VFS 多态。

> TUI 由 B 写,这一幕可由 **B 操作、你解说**,或你操作、B 补充。F 键若被 SSH/终端截获,用菜单提示的替代键,或退回 CLI `./pfs --fat16` 跑 FAT16 简版。

## 一页速记卡(打印这一页带去)

```
./pfs --format
pwd                                  # /
mkdir -p /home/alice ; tree
cd /home/alice ; pwd
touch notes.txt
open notes.txt rw                    # → fd=0  ← 念真实数字
write 0 Hello PseudoFS -- OS course demo
close 0 ; cat notes.txt
ll ; chmod 600 notes.txt ; ll        # rwxrwxrwx → rw-------
ln notes.txt hard.txt ; stat notes.txt   # Links: 2
ln -s notes.txt soft.txt ; readlink soft.txt ; ls   # soft.txt@
cp notes.txt copy.txt ; mv copy.txt renamed.txt ; ls
disk
useradd alice alice 1000 1000
su alice alice ; cat /home/alice/notes.txt   # permission denied ← 预期!
su root root
exit                                 # Disk saved.
./pfs                                # 不加 --format
cat /home/alice/notes.txt            # 数据还在 = 持久化
exit ; ./pfs --tui                   # F2 切FAT16 / F3 磁盘位图 / F10 退出
```

**翻车自救**:命令报错先看是不是 ① fd 念错 ② 写后用同 fd 读 ③ 漏了 `su` 密码 ④ 该不该带 `--format`。实在卡住:`exit` → `./pfs --format` 重开,从最近一个 Act 续。
