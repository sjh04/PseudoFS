# PseudoFS 现场演示脚本(TUI 全程版,主操作手:A)

> **全程在 TUI 里演示**——命令在终端面板里敲,左侧目录树、右侧磁盘条**每条命令后自动刷新**(无需手动 F5)。
> 命令分发和 CLI 完全一致(同一套 `CommandRegistry`),下方「→」是实测真实输出。
> 设计目标:10~12 分钟覆盖**全部 35 条命令 + 所有评分点**,开场即用**预装的 293KB 大文件**展示三级混合索引。
> 每个 Act 末标注它**证明了哪个功能点 / 必答题**。文末有「命令覆盖对照表」核对不漏命令。

## 0. 开演前 prep(90 秒,务必先做)

大文件没法在 TUI 里现场敲,**先把它灌进 TUI 的演示盘**:

```bash
cd PseudoFS/build && cmake .. && make -j$(nproc)        # 确认能编出 pfs

# 清掉旧盘(含改版前的 512 块旧盘),用 CLI 生成带 big.txt 的 UNIX 盘
rm -f pfs_unix.img pfs_tui_unix.img pfs_tui_fat16.img pfs_ops.log users.dat
python3 -c "print('touch /big.txt');print('open /big.txt w');[print('write 0 '+'B'*1000) for _ in range(300)];print('close 0');print('exit')" | ./pfs --format

# 把它作为 TUI 的 UNIX 演示盘(两种模式镜像格式相同,只是文件名不同)
cp pfs_unix.img pfs_tui_unix.img
```
> 这写出一个 **300000 字节(≈293KB)** 的 `/big.txt`。TUI 的 FAT16 盘已删除,启动时会自动新建一张干净的 2082 块 FAT16 盘,F2 照样能切。

**另开一个终端备用**(演 Act 1 的 inode 铁证用):
```bash
cd PseudoFS/build
# 这条 dump big.txt(inode 4)的地址数组,Act 1 现场或贴 PPT 用
python3 -c "
import struct
d=open('pfs_tui_unix.img','rb').read(); di=d[2*512+256:2*512+256+64]
a=struct.unpack_from('<10H',di,24)
print('addr[0..5] 直接   :',a[0:6])
print('addr[6]    一次间址:',a[6]); print('addr[7]    一次间址:',a[7])
print('addr[8]    二次间址:',a[8],'  <== 已启用!')
"
```

**环境**:终端拉大到至少 **100×30**(四面板才不挤)。

> ⚠️ 三个最易翻车点,先记牢:
> 1. **`open` 返回的 fd 是 `0`,不是 3**。后面 `write`/`close` 跟着屏幕上的真实数字念。
> 2. **`open` 不会自动建文件**——必须先 `touch` 再 `open`;写完别用同一 fd 读(偏移在末尾),要 `close` 后 `cat` 或重新 `open ... r`。
> 3. **Act 8 的 `permission denied` 是预期结果**——主动说"看,权限系统把它拦住了"。

## Act 1 — 开场亮点:大文件 + 三级混合索引(A-02 加分核心,必答题 Q4)★

> 在终端面板里敲:

```
ls /
stat /big.txt
disk
```
→ `ls /` → `./  ../  etc/  home/  big.txt`
→ `stat /big.txt` → `Size: 300000 bytes`
→ `disk` → `Blocks: 596/2048 used` · `Inodes: 5/256 used`

按 **F3** → 全屏磁盘块位图(红=已用 / 青=元数据 / 绿=空闲):**前 34 块青色元数据,紧接着一大片红色——正是这个大文件占的 ~590 块**。看完按任意键回主界面。

**inode 铁证**(切到备用终端,或贴 PPT):
```
addr[0..5] 直接   : (2044, 2043, 2042, 2041, 2040, 2039)
addr[6]    一次间址: 2038
addr[7]    一次间址: 1782
addr[8]    二次间址: 1525   <== 已启用!
```

**话术**:「一个 inode 的地址数组 `addr[10]`:前 6 项直接指数据块(≤3KB);`addr[6]`、`addr[7]` 各指一个一次间址块,每块再装 256 个块号(各覆盖 128KB);`addr[8]` 是二次间址,指向'指针块的指针块'。这个 293KB 文件**三级全用上了**——`addr[8]` 非零,说明它跨过了第 518 个数据块,真正走进二次间址。
块账也对得上:586 数据块 + 4 间址块 + 初始 6 块 = 596,和 `disk` 一致。
顺便:`addr` 里都是高号块(2044、2043…),因为成组链接法的 balloc 先发内存栈里的高号块——这是它和位图顺序分配的行为差异。」

> 答辩 Q4「哪些因素影响文件最大长度」:块大小(512B)、地址数组项数(10)、间址级数(直接/一次/二次)、数据区总块数(2048)——四个全在这一幕里看得见。

**证明**:A-02 混合索引(直接+一次+二次间址)· 必答题 Q4 · 大文件支持 · B-04 磁盘可视化(F3 位图)。

## Act 3 — 文件 I/O + 打开文件表

```
touch notes.txt
open notes.txt rw
write 0 Hello PseudoFS -- OS course demo
close 0
cat notes.txt
```
→ `open` → **`fd=0`**(念这个数字!)
→ `write 0 ...` → `32 bytes written`
→ `cat` → `Hello PseudoFS -- OS course demo`

**话术**:「`touch` 建空文件;`open` 在打开文件表分配 fd(这里 0),`write` 通过它定位 inode、按混合索引落盘,`close` 释放 fd。右边磁盘条又涨了一格。」
**证明**:E-01 touch · E-02 open · E-03 read · E-04 write · E-05 close · 打开文件表。

## Act 4 — 权限(rwx 访问控制)

```
chmod 600 notes.txt
ll
```
→ `ll` → `-rw-------  1 root  32  ...  notes.txt`(原本 `-rwxrwxrwx`)

**话术**:「权限位存在 inode 的 mode 里;`chmod 600` 后 owner 只读写、group/other 全无——Act 8 用它演真正的访问拦截。」
**证明**:A-05 文件访问控制 · C-01 chmod · D-05 ll。

## Act 5 — 链接(UNIX 引擎亮点,FAT16 做不到)

```
ln notes.txt hard.txt
stat notes.txt
ln -s notes.txt soft.txt
readlink soft.txt
ls
```
→ `stat notes.txt` → `Links: 2`(硬链接让 nlink 变 2)
→ `readlink soft.txt` → `notes.txt`
→ `ls` → `./  ../  notes.txt  hard.txt  soft.txt@`(`@` 标记软链接)

**话术**:「硬链接 `hard.txt` 和 `notes.txt` **指向同一 inode**,nlink=2、共享数据;软链接 `soft.txt` 是另一个 inode,存的是目标路径字符串。这是 UNIX 引擎比 FAT16 强的地方——FAT16 不支持链接,等下 F2 切过去就没有这功能。」
**证明**:A-04 硬/软链接 · B-03 stat · D-04 ls。

## Act 6 — 复制 / 移动 / 通配符 / 查找

```
cp notes.txt copy.txt
mv copy.txt renamed.txt
ls *.txt
find / notes.txt
disk
```
→ `cp` → `Copied.` · `mv` → `Moved.`
→ `ls *.txt` → `hard.txt  notes.txt  renamed.txt  soft.txt@`(通配符展开)
→ `find / notes.txt` → `/home/alice/notes.txt`
→ `disk` → `Blocks: 600/2048 used` · `Inodes: 9/256 used`

**话术**:「`ls *.txt` 是 shell 风格通配符展开;`find` 从指定目录递归按名搜索;`disk` 显示成组链管理下的用量——从开场的 596 涨到 600,大文件占了大头。」
**证明**:A-03 cp · B-01 mv · C-02 通配符 · B-06 find · B-04 磁盘可视化。

## Act 7 — 删除(more 分页器 / 空目录 / 递归)

```
more notes.txt
mkdir tmp
rmdir tmp
rm hard.txt
ls
```
→ `more notes.txt` → 进**全屏分页器**显示内容,任意键返回
→ `rmdir tmp` → 成功(空目录)
→ `rm hard.txt` → 成功
→ `ls` → `notes.txt  soft.txt@  renamed.txt`(hard.txt 没了,左侧树同步消失)

**话术**:「`more` 在 TUI 里是全屏分页器;`rmdir` 只删空目录;`rm` 删文件、`rm -r` 递归删目录树。删 `hard.txt` 只去掉一个目录项、把 `notes.txt` 的 nlink 减回 1,数据还在——这就是硬链接的意义。」
**证明**:B-02 more · D-02 rmdir · E-06 rm / rm -r。

## Act 8 — 多用户 + 访问控制拦截 + 改密/登录(全场高光)

```
useradd alice alice 1000 1000
su alice alice
cat /home/alice/notes.txt
su root root
```
→ `useradd` → `User alice created.`
→ `su alice alice` → `Switched to alice`(**标题栏 user 变 alice**)
→ `cat ...` → **`cat: /home/alice/notes.txt: permission denied`**
→ `su root root` → `Switched to root`

```
passwd root newroot
logout
login root newroot
```
→ `passwd` → `Password changed.` · `logout` → `Logged out.` · `login root newroot` → `Welcome, root (uid=0)`

**话术**:「`notes.txt` 是 root 的、权限 600。切到 alice 再读它——**被拒绝**:alice 不是 owner、gid 也不同,落到 other 位、无 r 权限。切回 root 绕过检查又能读。`passwd` 改密并落盘,`logout`/`login` 重新验证——账户表独立持久化在 `users.dat`。注意标题栏的 user 字段全程跟着变。」
**证明**:U-01 login · U-02 logout · U-03 useradd · U-04 passwd · B-05 su · A-05 权限检查(真实拦截 + root 绕过)。

## Act 9 — 操作日志与回放(C-05)★

```
history
log
```
→ `history` → 带行号的命令历史
→ `log` → 带时间戳和用户名的操作日志(只记真正改动 FS 的操作)

**话术**:「`history` 是命令历史(↑↓ 可翻阅);`log` 是操作日志,跨会话累积在 `pfs_ops.log`。它最强的用法是 `replay`——哪怕磁盘被格式化成空盘,也能按日志把整个目录结构重建出来(可在收尾时演:`./pfs --tui --format` 开空盘 → `replay` → 树面板里结构原样回来)。」
**证明**:C-03 命令历史 · C-05 操作日志 + 回放。

## Act 10 — 双引擎切换 F2(VFS 多态)

按 **F2** → 标题栏 `[UNIX]` 变 `[FAT16]`,左侧树切到 FAT16 盘(空盘)。在终端里:
```
mkdir d1
touch d1/f.txt
open d1/f.txt w
write 0 hello-fat16
close 0
cat d1/f.txt
```
→ `cat d1/f.txt` → `hello-fat16`

按 **F3** 看 FAT16 的块位图(元数据区更大——FAT 表扩到了 9 块)。再按 **F2** 切回 UNIX,大文件和刚才建的目录都还在。

**话术**:「F2 把底层从 inode 体系换成 FAT 表,**上层命令一行没改**——靠 `IFileSystem` 抽象接口,这就是 VFS 多态。两个引擎各有独立磁盘镜像,互不踩坏。FAT16 不支持链接和 rwx 权限,但基本文件操作完全一致。」
**证明**:A-01 双引擎 · VFS 多态 · B-04 磁盘可视化(两种布局对比)。

## Act 11 — 持久化(必考 F-03)

按 **F10** 退出 → `Disk saved.`(UNIX/FAT16 各自 dump 回自己的镜像)
```bash
./pfs --tui                 # 再次启动,不加 --format
```
→ 左侧树里 `big.txt`、`/home/alice/notes.txt` 等**原样还在**;在终端 `cat /home/alice/notes.txt` → `Hello PseudoFS -- OS course demo`

**话术**:「F10 退出时把虚拟磁盘 dump 成镜像;重启**不格式化**,从镜像 mount 恢复,数据原样还在。这就是持久化。」
**证明**:F-03 持久化。
